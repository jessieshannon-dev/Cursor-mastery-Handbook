This website is a beginner-friendly training guide for learning Cursor from scratch and understanding the full modern app-building workflow.

1. What Cursor Is

Cursor is an AI-powered code editor. It looks and feels a lot like VS Code, but it has AI built directly into the workflow.

Cursor is not the same thing as Lovable. Lovable is more like a guided app generator where you describe an app and it builds inside its own platform. Cursor is a real code editor where you own the actual codebase.

ToolBest ForMain Difference
LovableFast prototypes and app ideasMore guided and platform-based
CursorReal custom software projectsYou own and control the code
VercelHosting websitesDeploys your code from GitHub
SupabaseBackend and databaseStores app data, users, files, and auth
Simple explanation: Cursor is where you build. GitHub stores the code. Vercel hosts the website. Supabase stores the data. Porkbun owns the domain.

2. Install Cursor on Windows

  1. Go to cursor.com.
  2. Click Download for Windows.
  3. Open the installer after it downloads.
  4. Accept the default installation settings.
  5. Open Cursor.
  6. Sign in with GitHub, Google, or email. GitHub is recommended if you plan to deploy apps.

After Cursor Opens

  • Look for the Explorer panel on the left.
  • Look for the Terminal at the bottom.
  • Use Ctrl + L to open AI Chat.
  • Use Ctrl + Shift + P to open the Command Palette.

3. Install Git

Git tracks changes in your code. GitHub stores those changes online.

  1. Go to git-scm.com/downloads.
  2. Download Git for Windows.
  3. Run the installer.
  4. Use the default settings unless you know exactly what you are changing.
  5. Restart your computer if needed.

Verify Git Installed

git --version

If you see a version number, Git is installed.

4. Install Node.js

Node.js lets you run modern JavaScript and web apps on your computer.

  1. Go to nodejs.org.
  2. Download the LTS version.
  3. Install using the default settings.

Verify Node and npm

node -v
npm -v

If both commands return version numbers, you're good. Tiny victory parade, no confetti cannon required.

Do not blindly accept everything AI creates. Review, test, and ask questions. AI is powerful. It is also occasionally a confident little chaos goblin.

7. Cursor Chat vs Agent vs Inline Edit

Chat

Use Chat when you want explanations, planning, debugging help, or small code snippets.

Agent

Use Agent when you want Cursor to create or modify multiple files.

Inline Edit

Use Inline Edit when you highlight code and want a specific change.

Rule of thumb: Chat is for thinking. Agent is for building. Inline Edit is for fixing one spot.

After Cursor creates the project files, open the Terminal.

Then open the local address shown in the terminal, usually:

10. Git and GitHub From Scratch

Git is the change tracker. GitHub is the online storage place for your code.

First-Time Commands

git init
git add .
git commit -m "Initial commit"

Connect to GitHub

git remote add origin YOUR_GITHUB_REPOSITORY_URL
git branch -M main
git push -u origin main

Everyday Git Workflow

git add .
git commit -m "Describe what changed"
git push

11. Deploy to Vercel

  1. Go to vercel.com.
  2. Sign in with GitHub.
  3. Click New Project.
  4. Select your GitHub repository.
  5. Add environment variables if needed.
  6. Click Deploy.
Every time you push to GitHub, Vercel automatically rebuilds and redeploys the website.

12. Set Up Supabase

Supabase gives your app a backend without you having to build everything from scratch.

Supabase Can Handle

  • User accounts and login
  • Database tables
  • File uploads and images
  • Permissions
  • APIs
  • Realtime updates

Environment Variables

NEXT_PUBLIC_SUPABASE_URL=your_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key
Never expose the service role key publicly. That key is powerful. Treat it like the keys to the snack cabinet, except worse.

13. Connect a Porkbun Domain

  1. Buy a domain in Porkbun.
  2. Go to your project in Vercel.
  3. Open Settings → Domains.
  4. Add your domain.
  5. Vercel will show the DNS records you need.
  6. Go to Porkbun DNS settings.
  7. Add or update the records Vercel gave you.
  8. Wait for DNS propagation.

Porkbun owns the domain. Vercel hosts the site. Don't make this harder than it is; the internet already has enough nonsense.

15. Debugging With Cursor

When something breaks, do not panic. Copy the error and make Cursor earn its snacks.

Error-Fixing Prompt

I am getting this error. Explain what it means in plain English, identify the most likely cause, fix the code, and give me exact steps to test the fix. Do not make unrelated changes.

Common Problems

ProblemLikely CauseFix
npm not recognizedNode.js not installed or terminal needs restartInstall Node LTS and restart
git not recognizedGit not installed or PATH issueInstall Git and restart
Vercel build failedMissing env vars or TypeScript errorCheck build logs
Supabase login failsWrong URL/key or redirect settingsCheck env vars and auth URLs

17. QA Checklist Before Launch

  • Can a new user sign up?
  • Can a user log in and log out?
  • Are protected pages blocked when logged out?
  • Can inventory items be created?
  • Can inventory items be edited?
  • Can inventory items be deleted?
  • Does search work?
  • Does it work on a phone?
  • Does the Vercel build pass?
  • Are environment variables set in Vercel?
  • Are Supabase permissions secure?
  • Are there no fake buttons or dead links?
